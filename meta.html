sobre este site // condensando a nuvem

indivíduos e organizações seguem entregando seus dados e operações para a nuvem. diz-se tendência. se Bauman enxergou repetidamente liquidez em aspectos diversos da vida contemporânea, tal tendência leva a metáfora um passo além – o vapóreo, gasoso.

e embora a hiper-desenhada conveniência destes serviços siga persuadindo-nos do contrário, esta nuvem não significa uma libertação informacional da materialidade de estruturas computacionais – um universo de bits desencarnados de seus corpos de silício.

pelo contrário, a infra-estrutura física segue presente e crescente. tem peso e temperatura, e consome grandes quantidades de energia. só está cada vez menos dentro de nossas casas – menos sujeitas a nosso agenciamento individual.

na geografia da "nuvem", o digital não flutua no ar tanto quanto percorre as profundezas dos oceanos

inevitavelmente, o controle sobre as máquinas que suportam a rede mundial de computadores reproduz (e acentua) as desigualdades presentes no mundo; é, portanto, do domínio da política. a metáfora utópica da internet como rede totalmente descentralizada se desmancha frente ao mapa geográfico de sua infra-estrutura. o tráfego se concentra em nodos de grande influência sobre o fluxo da informação.

para construir estruturas de conexão mais distribuídas e que reflitam as múltiplas realidades culturais de populações alternas, alguns grupos propõem o aprimoramento e a utilização de tecnologias do tipo p2p (peer-to-peer), em que os usuários se conectam diretamente; em contraste com arquiteturas do tipo cliente-servidor, em que as conexões entre os usuários são mediadas por um sistema centralizado.

este movimento demonstra uma atitude de soberania e agenciamento sobre seus próprios dados, e se apoia em diversas iniciativas de software livre e aberto para o gerenciamento de protocolos de conexão e criptografia. o objetivo é ter redes mais democráticas e livres, resistentes a ações arbitrárias de agentes centralizados, como a censura (ou o impulsionamento arbitrário) de conteúdos ou a negociação de dados pessoais para a utilização no marketing.

somada a essa camada de software, também se coloca a necessidade de retomar o controle sobre a estrutura material que suporta a vida digital – os computadores onde estão guardadas todas essas informações. por isso, inspirado no projeto NODE NANO SERVER V2, construí este website a partir de linguagem simples html e css (você está convidado a consultar o código fonte) e preparei a estrutura para hospedá-lo em minha própria casa, utilizando um computador de placa única (SBC) de baixo custo e baixo consumo de energia.

======> você está aqui

um raspberry pi 0 w integra apenas as funções essenciais para executar o software do servidor: processamento, memória volátil e conectividade via wi-fi. o armazenamento dos arquivos do site fica sob responsabilidade de um cartão de memória micro-sd inserido na placa. o conjunto se finaliza com um case protetor feito de acrílico cortado a laser e uma fonte chaveada de 5 volts modificada para alimentar a placa diretamente pelos pinos GPIO.

todo o software utilizado é de código aberto: na base da pilha está o sistema operacional raspberry OS (uma distribuição do debian linux específica para as placas raspberry pi) na versão lite, que não inclui interface gráfica. as requisições http para o servidor são gerenciadas pelo nginx, que está instalado num container docker quarentenado no sistema.

como a minha conexão doméstica não permite um IP externo fixo (obrigado vivo fibra), para que você consiga acessar este site sempre utilizando a mesma URL é necessário o duck DNS, um serviço externo que reconfigura em tempo real o domínio sempre que o IP do servidor muda, mantendo o apontamento correto.

por fim, já que se trata de um sistema headless (que não possui tela ou dispositivos de entrada como teclado e mouse), todas as operações de instalação, configuração e manutenção são feitas em linha de comando via conexão ssh, a partir de outro computador conectado na mesma rede.

comparada ao uso de um servidor web convencional, esta arquitetura não traz nenhuma vantagem técnica. na verdade, em quase todos os aspectos deve haver desempenho bem abaixo do esperado em qualquer serviço comercial de hospedagem de sites. este site provavelmente não suportará mais de uma dezena de acessos simultâneos.

no entanto, além do intenso aprendizado e da economia com as mensalidades da hospedagem e domínio, este projeto vislumbra, mesmo que por uma pequena fresta, a possibilidade prática de tomar o agenciamento sobre os dados que disponibilizamos na internet. também traz ao consciente a realidade material deste objeto virtual específico, o site, solidificado em um corpo físico que demanda algum esforço de construção e manutenção.

como prova de conceito ou objeto apreciativo, este servidor continuará a ser atualizado e mantido por mim até o limite de meus esforços. de fato é este site encarnado em dispositivo eletrônico – se você está lendo essa página, é porque a placa raspberry pi está ligada na tomada e conectada na rede wi-fi de minha casa, rodando todo o software corretamente. esta minúscula parcela da internet, então, está desmistificada, solidificada em realidade tangível; portanto, neste nível mais básico (do hardware), sujeita a meu agenciamento direto.
